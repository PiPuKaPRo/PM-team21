# Оценка проекта
По итогам [Фазы 1](./phase_1.md) был выбран проект - **Мобильная/ПК игра в жанре приключений**.

**Концептуальное описание проекта:** Игра в жанре приключений, где игрокам предстоит выживать, коммуницировать и узнавать сюжет мира, адаптированная под мобильную версию.

## Use Cases
Далее приведены варианты использования приложения. 

### 1. Регистрация игрока
**Предусловия:** 
- Пользователь впервые запускает игру.
- Пользователь выбрал пункт «Регистрация».

**Основной поток:**
1. Пользователь вводит email и пароль.
2. Система проверяет что email ещё не зарегистрирован и пароль соответствует требованиям безопасности.
3. Система отправляет код подтверждения на email.
4. Пользователь вводит код подтверждения.
5. Система предлагает ввести имя персонажа.
6. Пользователь подтверждает ввод.

**Результат:**
Создан игровой аккаунт и персонаж пользователя.


### 2. Создание нового персонажа
**Предусловия:** 
- Пользователь зарегистрирован и авторизован.
- Пользователь начал новую игру.

**Основной поток:**
1. Система предлагает выбрать внешний вид персонажа.
2. Пользователь выбирает пол, базовую внешность и имя персонажа.
3. Пользователь подтверждает создание персонажа.

**Результат:**
Создан игровой персонаж и сохранено начальное состояние игры.


### 3. Исследование игровой локации
**Предусловия:** 
- Пользователь находится в игровом мире.

**Основной поток:**
1. Игрок перемещается по текущей локации.
2. Система отображает доступные интерактивные объекты и персонажей.
3. Игрок взаимодействует с объектами (осмотр, подбор, активация).

**Результат:**
Игрок исследовал локацию и получил новую информацию или ресурсы.


### 4. Взаимодействие с неигровым персонажем (NPC)
**Предусловия:** 
- Игрок находится рядом с NPC.

**Основной поток:**
1. Игрок инициирует диалог с NPC.
2. Система отображает реплики NPC.
3. Игрок выбирает вариант ответа.
4. NPC реагирует в зависимости от выбора игрока.

**Альтернативные потоки:**
- (3а) Игрок отказывается от диалога — диалог завершается.

**Результат:**
Продвижение сюжета или получение задания/информации.


### 5. Получение задания (квеста)
**Предусловия:** 
- Игрок взаимодействует с NPC или объектом задания.

**Основной поток:**
1. Система предлагает новое задание.
2. Игрок принимает задание.
3. Задание добавляется в журнал квестов.

**Результат:**
Активное задание добавлено в список текущих квестов.


### 6. Выполнение задания
**Предусловия:** 
- У игрока есть активное задание.

**Основной поток:**
1. Игрок выполняет действия, необходимые для задания (исследование, бой, диалог).
2. Система отслеживает прогресс задания.
3. Условия задания выполнены.

**Результат:**
Задание отмечено как выполненное.


### 7. Получение награды за задание
**Предусловия:** 
- Задание выполнено.

**Основной поток:**
1. Игрок возвращается к NPC или активирует завершение задания.
2. Система выдает награду (опыт, предметы, сюжетное продвижение).

**Результат:**
Игрок получил награду, прогресс персонажа обновлён.


### 8. Управление инвентарём
**Предусловия:** 
- У игрока есть предметы.

**Основной поток:**
1. Игрок открывает инвентарь.
2. Система отображает список предметов.
3. Игрок выбирает предмет для использования, экипировки или удаления.

**Результат:**
Состояние инвентаря обновлено.


### 9. Бой с противником
**Предусловия:** 
- Игрок столкнулся с противником.

**Основной поток:**
1. Система инициирует боевой режим.
2. Игрок выбирает действия (атака, защита, предмет).
3. Противник выполняет ответные действия.
4. Бой продолжается до победы или поражения.

**Результат:** 
Бой завершён, состояние персонажа обновлено.


### 10. Развитие персонажа
**Предусловия:**
- Игрок получил опыт или очки развития.

**Основной поток:**
1. Игрок открывает меню развития.
2. Система показывает доступные улучшения.
3. Игрок выбирает улучшение характеристик или навыков.

**Результат:** 
Характеристики персонажа улучшены.


### 11. Сохранение прогресса игры
**Предусловия:** 
- Игра запущена.

**Основной поток:**
1. Игрок выходит из игры или достигает контрольной точки.
2. Система автоматически сохраняет текущее состояние.

**Результат:**
Прогресс игры сохранён.


### 12. Загрузка сохранённой игры
**Предусловия:**
- У пользователя есть сохранения.

**Основной поток:**
1. Пользователь запускает игру.
2. Выбирает сохранённый слот.
3. Система загружает состояние игрового мира.

**Результат:** 
Игрок продолжает игру с сохранённого момента.


### 13. Просмотр сюжета и мира игры
**Предусловия:**
- Игрок открыл внутриигровое меню.

**Основной поток:**
1. Игрок заходит в раздел «Лор» или «Журнал».
2. Система отображает открытые записи о мире и сюжете.
3. Игрок выбирает запись для просмотра.

**Результат:** 
Игрок ознакомился с историей мира и сюжетом.
