
# Фаза 1: Выбор концепции проекта
## Задания
- Придумать не менее 4 концепций программного проекта.
- Сформулировать не менее 4 критериев (критерии могут характеризовать этот воображаемый продукт/проект, а могут вашу будущую работу по предмету над этой концепцией).
- Оценить важность критериев и альтернативы по критериям экспертным путём (каждому). 3 оценки упорядочить и рассматривать как пессимистичная, ожидаемая и оптимистичная.
- Определить наиболее подходящий вариант методом нечёткого коллективного TOPSIS.
- Определить наиболее подходящий вариант методом чёткого коллективного МАИ (использовать в качестве чётких оценок мат. ожидание трёхточечных).
- Проанализировать результаты, сделать выводы.

## Конецепции
В результате обсуждения, было выбрано 4 концепции:

1. **Единая платформа для управления умным домом** — Мобильное приложение, объединяющее устройства разных производителей через единое API. Поддерживает управление освещением, датчиками, камерами, климатом и сценариями автоматизации.
2. **Мобильная/ПК игра в жанре приключений** — Сюжетная игра с карточными механиками, различными ролями, сетевой игрой, прогрессией персонажей и соревновательным элементом.
3. **Телеграм-бот для подбора книг и учебных материалов** — Бот анализирует интересы пользователя и даёт рекомендации по книгам, статьям, учебным курсам. Возможен поиск по жанрам, авторам, темам и рейтингам.
4. **Веб-приложение для управления финансами в разных банках** — Платформа, агрегирующая счета пользователя из нескольких банков, позволяющая отслеживать транзакции, бюджеты, задолженности и финансовые цели.

## Критерии
Для выбора предпочтительной концепции (альтернативы) были сформулированы 5 критериев:
1. Количество похожих предложений (Чем ниже конкуренция - тем лучше)
2. Срок окупаемости проекта (Чем меньше срок - тем лучше)
3. Кол-во потенциальных клиентов
4. Наличие конкурентных преимуществ
5. Стоимость реализации MVP* (Чем меньше стоимость - тем лучше)

> *MVP (англ. minimum viable product, "минимально жизнеспособный продукт") - тестовая версия продкута с минимально необходимым набором функций.
